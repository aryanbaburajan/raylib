Some other conventions to follow\+:
\begin{DoxyItemize}
\item {\bfseries{ALWAYS}} initialize all defined variables.
\item {\bfseries{Do not use TABS}}, use 4 spaces instead.
\item Avoid trailing spaces, please, avoid them
\item Control flow statements always are followed {\bfseries{by a space}}\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{keywordflow}{if} (condition) value = 0;}
\DoxyCodeLine{}
\DoxyCodeLine{\textcolor{keywordflow}{while} (!WindowShouldClose())}

\DoxyCodeLine{}
\DoxyCodeLine{\textcolor{comment}{// Be careful with the switch formatting!}}

{\bfseries{If proposing new functions, please try to use a clear naming for function-\/name and functions-\/parameters, in case of doubt, open an issue for discussion.}}\hypertarget{md__c_o_n_v_e_n_t_i_o_n_s_autotoc_md15}{}\doxysection{Files and Directories Naming Conventions}\label{md__c_o_n_v_e_n_t_i_o_n_s_autotoc_md15}

\begin{DoxyItemize}
\item Directories will be named using {\ttfamily snake\+\_\+case}\+: {\ttfamily resources/models}, {\ttfamily resources/fonts}
\item Files will be named using {\ttfamily snake\+\_\+case}\+: {\ttfamily main\+\_\+title.\+png}, {\ttfamily cubicmap.\+png}, {\ttfamily sound.\+wav}
\end{DoxyItemize}

{\itshape NOTE\+: Avoid any space or special character in the files/dir naming!}\hypertarget{md__c_o_n_v_e_n_t_i_o_n_s_autotoc_md16}{}\doxysection{Games/\+Examples Directories Organization Conventions}\label{md__c_o_n_v_e_n_t_i_o_n_s_autotoc_md16}

\begin{DoxyItemize}
\item Data files should be organized by context and usage in the game, think about the loading requirements for data and put all the resources that need to be loaded at the same time together.
\item Use descriptive names for the files, it would be perfect if just reading the name of the file, it was possible to know what is that file and where fits in the game.
